# Апартаменты в Сочи — лендинг (React)

Одностраничный адаптивный сайт для аренды апартаментов на побережье Сочи, реализованный на React.  
Клиентская логика реализована на **React + Vite**: модальное окно, валидация формы, управление состоянием через хуки. Проект создан с фокусом на современные стандарты верстки, доступность, производительность и компонентный подход.

## Особенности

### Реализация на React
- **Компонентная архитектура**: проект разбит на переиспользуемые компоненты (`Header`, `Hero`, `Stats`, `ApartmentsGrid`, `Modal`, `Footer`)
- **Управление состоянием через хуки**: `useState` для контроля видимости модального окна, данных формы и валидации
- **JSX-синтаксис**: декларативное описание интерфейса вместо прямого манипулирования DOM
- **Локальное состояние**: данные формы и статусы валидации хранятся в состоянии компонента, а не в глобальной области видимости

### Интерактивная форма
-   Модальное окно открывается по кнопке «Выбрать апартаменты»
-   Поля: даты заезда/выезда, количество гостей, телефон, комментарий
-   Валидация в реальном времени с обратной связью пользователю
-   Закрытие по кнопке ×, клику на оверлей и клавише Esc
-   Автоматическая очистка формы после отправки
-   *Важно: вся логика реализована через React-хуки, без прямых обращений к DOM*

###  Вёрстка и стили
-   Семантическая HTML-разметка (`<header>`, `<section>`, `<article>`, `<address>`)
-   Полная адаптивность (от 320px до 4K)
-   Архитектура: БЭМ + CSS-переменные
-   Доступность: `aria-label`, `alt` для изображений, поддержка клавиатурной навигации

### Производительность
-   Локальные шрифты (Oswald с кириллицей) в формате WOFF2
-   Изображения в формате WebP
-   Lazy loading для изображений
-   Отсутствие внешних зависимостей (все ресурсы локальные)
-   Сборка через **Vite**: быстрый дев-сервер, оптимизированный продакшен-билд
-   Lighthouse оценки для Mobile: в категориях Performance - 98, Accessibility - 94, SEO - 96, Best Practices - 91.

###  Качество кода
-   Валидный код: проходит проверку [W3C Validator](https://validator.w3.org/)
-   SEO-оптимизация: мета-описание, семантические заголовки, корректные `base` пути для GitHub Pages
-   Настроенный `.gitignore` для исключения служебных папок (`node_modules`, `dist`)

## Зачем это всё
Проект создан для портфолио frontend-разработчика. 

Практиковала react, училась переносить готовый код на компонентную архитектуру. Разбиралась с базовыми концепциями: JSX, useState, управление событиями,сборка (Vite). Изучала, когда есть смысл использовать React, а когда это избыточно (как в моем случае).
